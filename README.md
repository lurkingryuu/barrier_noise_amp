# Barrier Noise Amplification in Cloud MPI Jobs

**CloudSim Plus-based simulation study of how cloud VM performance variability amplifies
makespan and idle waste in MPI barrier-synchronized parallel jobs.**

Course project — High Performance Cloud Computing (B.Tech, March 2026)

---

## Project Overview

Tightly coupled MPI applications rely on global `MPI_Barrier` synchronization between
compute phases. In cloud environments, VM performance variability (noisy neighbors,
hypervisor jitter, network latency) causes stragglers that force all other ranks to idle.
This project quantifies the amplification effect as process count, noise magnitude, and
noise distribution vary.

**Key results (B=100 phases, ε=0.10):**
| Noise Model | P=8 | P=64 | P=256 |
|-------------|-----|------|-------|
| No Noise    | 0.000 | 0.000 | 0.000 |
| Gaussian    | 0.141 | 0.234 | 0.282 |
| Lognormal   | 0.148 | 0.258 | 0.319 |
| Bursty      | 0.171 | 0.555 | 0.662 |

*(Amplification factor A = (M − M_ideal) / M_ideal)*

## Quick Start

```bash
# Clone / enter the repo
cd code/

# Build fat JAR
make build

# Run full experiment (384 configs × 30 trials)
make full

# Generate all 7 figures
make figures

# Run CloudSim Plus DES demo (real discrete-event simulation)
make demo

# Run CloudSim Plus DES vs. analytical validation sweep
make cloudsim
```

Or, without Make:
```bash
mvn package -DskipTests
java -jar target/barrier-noise-amplification-1.0.0.jar full
python3 analysis/plot_results.py
```

## Simulation Architecture

### CloudSim Plus DES Mode (`make demo` / `make cloudsim`)

Uses the real CloudSim Plus 8.5.7 discrete-event simulation engine:

```
CloudSimPlus (DES engine)
  └── DatacenterSimple (1 host, P PEs at 1000 MIPS each)
       └── P × VmSimple (1 PE, 1000 MIPS each)
            └── BarrierBroker (extends DatacenterBrokerSimple)
                 └── Phase loop:
                      1. Submit P cloudlets (noisy lengths: 1000 × (1+η) MI)
                      2. Count CLOUDLET_RETURN events
                      3. When all P done → record phase → submit next phase
```

**CloudSim Plus ↔ MPI mapping:**
| MPI Concept | CloudSim Plus Object |
|-------------|---------------------|
| MPI rank | `VmSimple` |
| Compute phase | `CloudletSimple` |
| Phase runtime | `cloudlet.length / vm.mips` |
| Noise injection | Modified cloudlet length |
| `MPI_Barrier` | `BarrierBroker.processCloudletReturn()` |

### Analytical Mode (`make full` / `make quick`)

Fast BSP engine for the 384-config × 30-run parameter sweep:
- Mathematically identical to the DES mode
- ~1000× faster (no discrete-event overhead)
- Validation: DES and analytical agree to < 0.01 % on makespan

### Noise Models

| Model | Distribution | Cloud Motivation |
|-------|-------------|-----------------|
| NONE | η = 0 | Ideal dedicated cluster |
| GAUSSIAN | η ~ N(0, ε²) | General CPU/memory jitter |
| LOGNORMAL | Zero-mean lognormal, σ = √ln(1+ε²) | Cloud latency fits (Underwood et al. ICPE 2018) |
| BURSTY | 5 % spike at 5ε; else ~0 | Noisy-neighbor interference (Ahuja et al. 2025) |
